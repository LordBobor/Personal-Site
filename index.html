Net By Net

Instream, Декабрь 2014 — по настоящее время

Участие в разработке личного кабинета интернет-провайдера Net By Net в комманде разработчиков.

-Кэширование с использованием библиотеки Robospice.
-push-уведомления.
-взаимодействия с Web-Service и парсинг JSON посредством библиотеки Retrofit.
-сложные кастомные компоненты UI(пятикратно раскрывающийся список).
-Использование системы контроля версий SVN.
-Использование багтрекера Atlassian Jira.

RVBuyer

Instream, Январь 2015 — по настоящее время

Доведение проекта RVBuyer, создаваемое для менеджеров по закумкам фирмы Rendez-Vous, до релиза.

-Рефакторинг и поддержка кода, последовательно прошедшего через руки 4 программистов.
-Исправление багов приложения.
-Работа с базой SQLite через ORM.
-Оптимизация работы с ORM.
-Работа с Web-Service.
-Генерация и парсинг XML.
-Работа над требованиям к доработкам непосредственно с заказчиком.
-Использование системы контроля версий SVN.
-Использование багтрекера Atlassian Jira.

Eristica

Start Industries, Апрель 2014 — Декабрь 2014

Разработка приложения Eristica, представляющее собой соцальную сеть для споров с друзьями.

-Работа в команде с другими разработчиками, тестерами и дизайнером.
-Работа по гибким методологиям разработки Scrum и Kanban.
-Разработка приложения с нуля до продакшена.
-Внедрение облачного бэкэнда на базе BaaS Parse.
-Кэширование контента с использованием Parse LocalDataStore.
-Опыт работы и интеграции приложения с соцсетями(VK, Facebook).
-Создание сложных кастомных UI компонент.
-Работа с загрузкой/выгрузкой видео с использованием ftp.
-Внедрение сложных анимаций и эффектов (Параллакс; блюр; интерактивная анимация; окно, выпадающее если потрясти телефон, итд...).
-Оптимизация списка с большим количеством фотографий для достижения плавного скролла.
-Создание анимированного туториала к приложению.
-Внедрение комментариев к спорам.
-Добавление возможности поделиться спором в соцсетях, снятие скриншота части экрана.
-Добавление в приложение аналитики средствами API Mixpanel.
-Сбор статистики по ANR средствами Crittercism.
-Использование систем контроля версий Git и Bitbucket.
-Использование багтрекера Atlassian Jira.

iParking

SIBIR LLC, Октябрь 2013 — Апрель 2014

Разработка приложения по поиску парковок в городе Москва.

Полный цикл разработки приложения.
-Проектирование и дизайн UX.
-Написания функциональных требований к приложению на основе пожеланий заказчика.
-Бумажное прототипирование.
-Создание первой версии дизайна.
-Работа с дизайнером по редизайну внешнего вида приложения.
-Работа с серверным разработчком над архитектурой данных приложения.
-Разработка приложения на основе макетов дизайна и функциональных требований.
    -Работа с API Яндекс Карт.
    -Работа с геолокацией.
    -Создания сложных кастомных компонент UI(Боковые меню, карточки парковок, прогрессбары, элементы анимации).
    -Фильтрация объектов на карте.
-Функциональное тестирование приложения.

Приложение не было опубликовано в Google Play в силу того, что компания обанкротилась и разработка приложения прекратилась.
